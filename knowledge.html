PC lent / performances dégradées

Symptômes

  • PC très lent / PC qui rame / Blocages fréquents / Ordinateur qui chauffe
  • Démarrage du PC très long / Ouverture de session lente / Applications longues à s'ouvrir
  • Disque à 100 % / Programmes ne répondent pas
  • Navigateur internet très lent
  • Ventilateur très actif

Description de la procédure

Cette procédure permet de diagnostiquer rapidement les causes les plus fréquentes de lenteur sur un poste de travail. Les vérifications suivantes consistent à contrôler l'état général du système (espace disque, utilisation des ressources, programmes actifs et mises à jour éventuelles) afin d'identifier un élément pouvant expliquer les ralentissements. L'objectif est de résoudre les cas simples au niveau support N1 ou de collecter les informations nécessaires pour une escalade vers un support technique supérieur si le problème persiste.

Work Note

Utilisateur signale des lenteurs sur son poste.

Vérifications effectuées :
- Redémarrage du poste
- Vérification espace disque
- Vérification utilisation ressources (CPU / RAM / Disque)

Captures jointes :
- Gestionnaire des tâches - Processus
- Gestionnaire des tâches - Performance
- Espace disque (Ce PC)

Vérifications

1Redémarrer le poste

Demander à l'utilisateur si un redémarrage a été effectué. Si non, redémarrer le PC et vérifier si les lenteurs persistent.

2Vérifier l'espace disque

  • Ouvrir Explorateur de fichiers
  • Aller dans Ce PC
  • Vérifier l'espace libre du disque C:

Si le disque est presque plein (moins de 10-15 % libre) → cela peut provoquer des lenteurs.

Faire une capture d'écran du disque C

3Ouvrir le Gestionnaire des tâches

  • Faire Ctrl + Maj + Échap
  • Aller dans l'onglet Processus

Observer :

  • CPU
  • Mémoire
  • Disque

Identifier si une ressource est proche de 100 %.

Faire une capture d'écran du Gestionnaire des tâches/Processus

Faire une capture d'écran du Gestionnaire des tâches/Performance

4Identifier un processus consommateur

Dans le gestionnaire des tâches :

  • Trier par CPU
  • Trier par Mémoire
  • Trier par Disque

Vérifier si un programme utilise une ressource de manière excessive.

5Vérifier les programmes au démarrage

  • Dans le Gestionnaire des tâches → onglet Démarrage
  • Identifier les programmes activés et impact élevé

Si nécessaire, désactiver les programmes non nécessaires

6Vérifier les mises à jour Windows

  • Ouvrir Paramètres
  • Aller dans Windows Update

Vérifier si :

  • Une mise à jour est en cours
  • Un redémarrage est requis

Les mises à jour peuvent provoquer des ralentissements temporaires.

8Vérifier l'activité antivirus

Vérifier si un scan antivirus est en cours, ce qui peut utiliser beaucoup de ressources.

9Nettoyer les fichiers temporaires

  • Ouvrir Nettoyage de disque ou Paramètres → Stockage
  • Supprimer les fichiers temporaires inutiles.

Cocher uniquement :

  • Fichiers temporaires
  • Miniatures
  • Fichiers d'optimisation de livraison
  • Cache DirectX Shader

10Tester les performances après actions

Demander à l'utilisateur d'ouvrir les applications concernées pour vérifier si les lenteurs persistent.
